# 📋 사장님비서 — MVP 프로젝트 계획서

---

## 1. 프로젝트 정의

### 1.1 미션

> 외식업 사장님의 매일 반복되는 운영 업무(마감 정리, 발주, 경비 관리, 리뷰 응대)를 AI로 자동화하여, 매일 밤 1시간 걸리던 마감 업무를 5분으로 줄여주는 올인원 웹앱

### 1.2 타겟 유저

| 구분 | 1차 타겟 | 2차 타겟 |
|------|----------|----------|
| 대상 | 개인 음식점 사장님 | 프랜차이즈 점주 |
| 규모 | 테이블 10~30개, 직원 0~3명 | 1~3개 매장 운영 |
| 특징 | 사장님이 직접 발주·마감·세무 처리 | 본사 시스템 외 자체 최적화 필요 |
| 페인포인트 | "체계적으로 하고 싶은데 시간 없어서 감으로 운영" | "내 매장 데이터를 내가 직접 분석하고 싶다" |

### 1.3 법적 안전 설계 (프랜차이즈 호환)

- **본사 시스템 연동 없음:** POS나 내부 시스템에 접속하지 않고, 사장님 직접 입력 데이터만 사용
- **영업 비밀 수집 없음:** 레시피, 본사 매뉴얼, 내부 가격 정보를 요구하지 않음
- **개인 경영 보조 도구:** 법적으로 사장님 개인의 경영 판단을 돕는 메모/분석 도구
- 엑셀이나 개인 메모장을 쓰는 것과 동일한 성격 → 프랜차이즈 계약 저촉 없음

### 1.4 Supabase 공유 환경

⚠️ 이 프로젝트의 Supabase는 다른 웹사이트들과 함께 공유하는 기존 프로젝트다.

- **기존 데이터 불가침:** 기존 테이블, RLS 정책, Storage 버킷 등을 절대 건드리지 않는다
- **테이블 프리픽스:** 사장님비서의 모든 테이블은 `sb_` 프리픽스 (예: `sb_stores`, `sb_daily_closing`)
- **auth.users 재활용:** 별도 users 테이블 생성 없이 기존 Supabase Auth 활용. 사장님비서 전용 프로필은 `sb_user_profiles`에 저장
- **Storage 프리픽스:** 영수증 이미지 등은 `sajang-receipts` 버킷에 저장
- **RLS 정책 프리픽스:** 정책 이름도 `sb_` 프리픽스로 기존과 구분
- **환경 변수:** 기존 Supabase URL/KEY를 그대로 사용

> 이 계획서에서 테이블명은 가독성을 위해 프리픽스 없이 표기하는 경우가 있으나,
> 실제 구현 시에는 반드시 `sb_` 프리픽스를 붙여야 한다.

### 모듈 ① 마감 리포트

#### 목적
사장님이 매일 영업 종료 후 30초 안에 매출을 기록하고, 자동으로 정리된 리포트를 받는 것

#### 입력 시스템 (3가지 모드)

**모드 A — 숫자패드 모드 (기본)**

```
┌─────────────────────────────┐
│  오늘의 매출 입력             │
│                             │
│  ┌─────────────────────┐    │
│  │    ₩ 1,870,000      │    │  ← 큰 숫자 표시
│  └─────────────────────┘    │
│                             │
│  ┌─────┬─────┬─────┐       │
│  │  1  │  2  │  3  │       │
│  ├─────┼─────┼─────┤       │
│  │  4  │  5  │  6  │       │  ← 큰 버튼 키패드
│  ├─────┼─────┼─────┤       │
│  │  7  │  8  │  9  │       │
│  ├─────┼─────┼─────┤       │
│  │ 00  │  0  │  ⌫  │       │
│  └─────┴─────┴─────┘       │
│                             │
│  채널별 매출 분배:            │
│  홀   ├────────●──┤  68%    │  ← 슬라이더 or 직접 입력
│  배민  ├───●───────┤  20%    │
│  쿠팡  ├──●────────┤   8%    │
│  포장  ├─●─────────┤   4%    │
│                             │
│  결제수단:                   │
│  카드 ├──────────●─┤  92%    │
│  현금 ├●───────────┤   8%    │
│                             │
│  메모 (선택): [오늘 비와서..] │
│                             │
│  [  💾 마감 저장  ]           │
└─────────────────────────────┘
```

- 채널 비율 조정 시 실시간으로 금액 자동 계산 표시
- "자주 쓰는 비율" 프리셋 저장 가능 (예: "평일 기본", "주말 기본")
- 이전 영업일 비율을 기본값으로 자동 세팅

**모드 B — 음성 모드**

- 마이크 버튼 탭 → "오늘 매출 백팔십칠만, 홀 백이십 배달 육십칠" 말하기
- AI가 파싱: { total: 1870000, 홀: 1200000, 배달: 670000 }
- 파싱 결과를 카드로 보여주고, 사장님이 확인 후 저장
- 한국어 숫자 변환 처리: "백팔십칠만" → 1,870,000 / "이백삼" → 2,030,000

**모드 C — 대화 모드**

- 채팅 입력창에 자유롭게 입력
- 예시: "오늘 187만 홀120 배67 비오는데 장사 잘됐음"
- AI가 구조화: 총매출 187만, 홀 120만, 배달 67만, 메모 "비오는데 장사 잘됐음"
- 누락된 정보는 추가 질문: "포장 매출은 따로 없으셨나요?"

#### 수수료 자동 계산 (실시간)

매출 입력 즉시 하단에 수수료 내역이 자동 표시:

```
┌─────────────────────────────────────┐
│  💰 오늘의 매출 분석                  │
│                                     │
│  총매출              ₩1,870,000     │
│  ├─ 홀              ₩1,200,000      │
│  │  └─ 카드수수료      -₩8,640      │
│  ├─ 배민              ₩520,000      │
│  │  ├─ 중개수수료     -₩35,360      │
│  │  └─ 배달대행       -₩29,700      │
│  ├─ 쿠팡이츠          ₩100,000      │
│  │  └─ 수수료         -₩9,800       │
│  └─ 포장               ₩50,000      │
│     └─ 카드수수료        -₩360       │
│  ─────────────────────────────      │
│  총 수수료            -₩83,860      │
│  실 수령액          ₩1,786,140      │
│  수수료 비율             4.5%        │
│                                     │
│  [총매출 기준 ○ ● 순매출 기준]  ← 토글 │
└─────────────────────────────────────┘
```

#### 수수료 포함/미포함 토글

- 화면 상단에 항상 표시되는 글로벌 토글 스위치
- **총매출 기준**: 모든 금액을 수수료 차감 전으로 표시
- **순매출 기준**: 모든 금액을 수수료 차감 후(실 수령액)으로 표시
- 이 토글은 앱 전체에 일괄 적용됨 (마감 리포트, 발주, 브리핑 등 모든 화면)
- Zustand 글로벌 스토어로 관리하여 모든 컴포넌트에서 참조

#### 자동 생성 리포트 항목

매출 저장 후 즉시 생성:

- 전일 대비 매출 증감률 (↑12% / ↓5%) + 금액
- 같은 요일 최근 4주 평균 대비 비교
- 홀/배달/포장 비율 변화 추이 (미니 차트)
- 날씨·요일 상관관계 자동 태깅 ("비 온 화요일 평균 대비 +8%")
- 채널별 수수료 합계 + 실 수령액
- 메모 AI 요약

#### 누적 데이터 뷰

- 일별/주별/월별 매출 그래프 (터치 시 상세 팝업)
- 요일별 평균 매출 히트맵 (어떤 요일이 강한지/약한지 한눈에)
- 월 목표 매출 설정 → 달성률 프로그레스바
- 전월/전년 동기 대비 성장률
- 채널별 매출 비중 추이 (배달 의존도 변화)

### 모듈 ② 발주 추천 시스템

#### 목적
식자재 발주를 "감"이 아닌 "데이터"로 결정하게 도와주는 것. 폐기 줄이고, 품절 막기.

#### 초기 세팅 (최초 1회, 5분 이내)

**Step 1 — 업종 선택**

```
우리 가게 업종을 선택해주세요:
[한식] [중식] [일식] [양식]
[카페] [분식] [치킨] [피자]
[베이커리] [주점] [기타 직접입력]
```

선택 시 해당 업종 기본 식자재 템플릿이 자동 로드됨

**Step 2 — 품목 리스트 편집**

```
┌───────────────────────────────────────────┐
│ 🥬 채소류                                  │
│ ☑ 양파      1망(15kg)    ₩12,000          │
│ ☑ 대파      1단          ₩3,500           │
│ ☑ 마늘      1kg          ₩8,000           │
│ ☐ 당근      (체크하여 추가)                 │
│ ☐ 감자                                    │
│ [+ 품목 추가]                              │
│                                           │
│ 🥩 육류                                    │
│ ☑ 삼겹살    1kg          ₩15,000          │
│ ☑ 목살      1kg          ₩14,000          │
│ [+ 품목 추가]                              │
│                                           │
│ 🧴 소스·양념류                              │
│ ...                                       │
└───────────────────────────────────────────┘
```

각 품목에 입력하는 정보:
- 품목명
- 단위 (kg / 팩 / 박스 / 개 / 망 / 판 / 병 / 포 등)
- 1단위 가격 (선택, 비용 계산용)
- 기본 발주량 (평소 1회 발주 시 주문량)
- 유통기한 (선택, 폐기 관리용)
- 거래처명 + 연락처 (선택)

**Step 3 — 카테고리 그룹핑**

기본 카테고리: 채소류 / 육류 / 수산류 / 양념·소스 / 음료·주류 / 유제품 / 냉동식품 / 소모품
사장님이 카테고리 추가·편집 가능

#### 일일 사용량 입력 (매일, 30초 목표)

**퀵 입력 화면:**

```
┌───────────────────────────────────────┐
│  📦 오늘 사용량 입력  (3/15 토)         │
│                                       │
│  프리셋: [평일기본] [주말기본] [직접입력]  │
│                                       │
│  🥬 채소류                              │
│  양파    [−] 1.5망 [+]    (재고 2.5망)  │
│  대파    [−]  2단  [+]    (재고 3단)    │
│  마늘    [−] 0.5kg [+]   (재고 2kg)    │
│                                       │
│  🥩 육류                                │
│  삼겹살  [−]  5kg  [+]    (재고 8kg)    │
│  목살    [−]  3kg  [+]    (재고 5kg)    │
│                                       │
│  [💾 저장] [🗑 폐기 입력]               │
└───────────────────────────────────────┘
```

**핵심 UX 기능:**
- **프리셋 탭:** "평일 기본" / "주말 기본" 선택 시 각 품목 평균 사용량 자동 채움. 미세 조정만 하면 됨
- **+/- 스테퍼:** 0.5 단위로 증감. 길게 누르면 빠르게 증가
- **재고 자동 계산:** 어제 재고 - 오늘 사용량 = 현재 잔여 재고 (자동 표시)
- **음성 입력:** "삼겹살 5키로, 양파 한 망 반, 콜라 한 박스" → 자동 인식
- **폐기 별도 입력:** 버린 식자재는 "폐기" 버튼으로 별도 기록. 사용량과 분리

#### AI 발주 추천

사용량 데이터가 1주일 이상 쌓이면 AI 추천이 활성화됨

**추천 화면:**

```
┌───────────────────────────────────────────┐
│  📊 내일 발주 추천  (3/16 일요일, 맑음)     │
│                                           │
│  예상 손님 수: 약 85명                      │
│  (근거: 최근 4주 일요일 평균 82명 + 맑음)    │
│                                           │
│  ⚠️ 부족 예상 (발주 필요)                   │
│  ┌─────────────────────────────────────┐  │
│  │ 삼겹살  재고 3kg → 내일 예상 6kg 필요  │  │
│  │         🔴 3kg 부족  추천: 5kg 발주   │  │
│  │         [−] 5kg [+]  [✓ 확정]        │  │
│  ├─────────────────────────────────────┤  │
│  │ 양파    재고 1망 → 내일 예상 1.5망     │  │
│  │         🟡 0.5망 부족  추천: 1망 발주  │  │
│  │         [−] 1망 [+]   [✓ 확정]       │  │
│  └─────────────────────────────────────┘  │
│                                           │
│  ✅ 재고 충분 (발주 불필요)                   │
│  대파 (재고 5단, 내일 예상 2단)              │
│  마늘 (재고 3kg, 내일 예상 0.5kg)           │
│                                           │
│  [ 📋 발주서 생성 ]                         │
└───────────────────────────────────────────┘
```

**발주서 생성:**

확정 버튼 누른 품목들을 거래처별로 묶어서 발주 메모 생성:

```
[거래처: 한마음 정육] 3/16 발주
- 삼겹살 5kg
- 목살 3kg
합계: ₩119,000

[거래처: 새벽 농산] 3/16 발주
- 양파 1망
- 대파 2단
합계: ₩19,000
```

→ "카카오톡 복사" / "문자 복사" 버튼으로 거래처에 바로 전송

**추천 알고리즘 로직:**

```
내일 예상 사용량 = 
  (최근 4주 같은 요일 평균 사용량)
  × (날씨 보정 계수: 비 -10%, 폭염 -15%, 한파 -10%, 맑음 +5%)
  × (매출 추세 보정: 최근 1주 매출이 4주 평균 대비 ±N%이면 비례 조정)

권장 발주량 = 
  max(0, 내일 예상 사용량 × 1.2(여유분) - 현재 재고)
  → 발주 단위(망, 박스 등)로 올림 처리
```

#### 분석 대시보드

- 품목별 월간 사용량 추이 그래프
- 식자재비 / 매출 비율 (원가율) — 총매출/순매출 토글 적용
- 폐기율 트래킹: 월별 폐기 금액 + 폐기 TOP5 품목
- 발주 적중률: AI 추천량 vs 실제 사용량 비교
- 품목별 단가 변동 기록 (수동 입력, 가격 오를 때 기록)

### 모듈 ③ 영수증 AI 경비 장부

#### 목적
영수증 사진만 찍으면 경비 장부가 자동 완성. 원하는 기간/조건으로 자유롭게 조회·다운로드.

#### 영수증 촬영·인식

**촬영 플로우:**

```
[📸 영수증 촬영] 탭
    ↓
카메라 열림 (또는 갤러리에서 선택)
    ↓
여러 장 연속 촬영 가능 (배치 모드)
    ↓
각 영수증 AI 자동 인식
    ↓
인식 결과 확인 화면
```

**AI 인식 항목:**

| 항목 | 설명 | 필수/선택 |
|------|------|-----------|

### 모듈 ④ 리뷰 답글 생성기

#### 목적

#### 초기 세팅 (톤 학습)

**Step 1 — 기존 답글 학습**

```
기존에 잘 쓴 답글 3~5개를 붙여넣어주세요.
AI가 사장님의 말투와 표현 스타일을 학습합니다.

[답글 2 붙여넣기]
...

[학습 시작 →]
```

**Step 2 — 매장 정보 등록**

- 매장명: "김사장 칼국수"
- 대표 메뉴: "손칼국수, 수제비, 녹두전"
- 매장 특징: "40년 전통, 직접 반죽, 할머니 레시피"
- 자주 쓰는 표현: "감사합니다~", "맛있게 드셨다니 기쁩니다"
- 이모지 사용: 사용함 / 안 함

**Step 3 — 톤 선택**

```
기본 톤을 선택해주세요:
[😊 친근한 동네 사장님]  → "감사해요~ 또 놀러오세요 ㅎㅎ"
```

## 3. 모듈 간 데이터 연동 맵

```
┌────────────┐     매출 데이터      ┌────────────┐
│  ① 마감     │ ─────────────────→ │  ② 발주     │
│  리포트     │     예상 손님 수     │  추천       │
└──────┬─────┘     계산에 활용      └──────┬─────┘
       │                                  │
       │ 매출·수수료                   식자재비│
       │ 데이터                       데이터  │
       ▼                                  ▼
┌────────────┐                    ┌────────────┐
│  💸 수수료   │                    │  ③ 영수증   │
│  분석       │                    │  경비 장부   │
└──────┬─────┘                    └──────┬─────┘
       │                                  │
       │      모든 데이터 종합              │
       └──────────┐  ┌────────────────────┘
                  ▼  ▼
            ┌────────────┐
            │  ⑤ 경영     │
            │  브리핑      │  ← ④ 리뷰 데이터도 여기로
            └────────────┘
```

- 마감 리포트 매출 → 발주 추천의 예상 손님 수 계산
- 마감 리포트 매출 → 수수료 분석의 채널별 수수료 계산
- 영수증 장부 식자재비 → 경영 브리핑의 원가율 계산
- 발주 시스템 폐기 데이터 → 경영 브리핑 효율 분석
- 리뷰 데이터 → 경영 브리핑 평판 카드
- 수수료 데이터 → 마감 리포트 순매출 표시 + 경영 브리핑 수수료 카드

## 4. 수수료 시스템 상세

### 4.1 수수료 설정 (초기 세팅)

**배달앱 수수료 등록:**

```
사용하는 배달앱을 선택해주세요:
☑ 배민 (중개 6.8%)        → 수수료율 수정 가능
☑ 쿠팡이츠 (9.8%)        → 수수료율 수정 가능
☐ 요기요 (12.5%)
☐ 땡겨요 (2.0%)

배달대행 수수료:
건당 [3,300]원  (또는 비율 [  ]%)
```

**카드 수수료 등록:**

```
연매출 구간을 선택해주세요:
○ 3억 이하 (영세: 0.5~0.8%)
● 3~5억 (중소: 1.3%)
○ 5~10억 (1.4%)
○ 10억 이상 (일반: 1.5~2.0%)

체크카드 비율: [20]%
→ 신용카드 수수료 1.3%, 체크카드 0.8% 자동 적용
```

**기타 수수료:**

```
[+ 수수료 항목 추가]
항목명: [네이버 주문]
유형: ● 비율(%)  ○ 건당 금액(원)
수수료: [3.5]%
```

### 4.2 계산 엔진 로직

```
입력: 총매출 ₩1,870,000
      홀 ₩1,200,000 / 배민 ₩520,000 / 쿠팡이츠 ₩100,000 / 포장 ₩50,000
      카드 92% / 현금 8%
      배민 배달 9건

Step 1) 배달앱 수수료
  배민: ₩520,000 × 6.8% = ₩35,360
  쿠팡이츠: ₩100,000 × 9.8% = ₩9,800

Step 2) 배달대행 수수료
  배민 9건 × ₩3,300 = ₩29,700

Step 3) 카드 수수료 (홀 + 포장만, 배달앱은 자체 정산)
  홀 카드매출: ₩1,200,000 × 92% = ₩1,104,000
  포장 카드매출: ₩50,000 × 92% = ₩46,000
  카드수수료: (₩1,104,000 + ₩46,000) × 1.3% = ₩14,950
  → 반올림: ₩14,950

Step 4) 합산
  총 수수료 = ₩35,360 + ₩9,800 + ₩29,700 + ₩14,950 = ₩89,810
  순매출 = ₩1,870,000 - ₩89,810 = ₩1,780,190
  수수료율 = 4.8%
```

### 4.3 수수료 포함/미포함 토글

- 글로벌 Zustand 스토어: `useFeeToggle` → `{ mode: 'gross' | 'net' }`
- 앱 상단 헤더에 항상 표시
- 모든 금액 표시 컴포넌트에서 이 스토어를 참조하여 총매출/순매출 전환
- 토글 변경 시 localStorage에 저장하여 다음 접속에도 유지
